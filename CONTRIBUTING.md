# Contributing to MovieMate

Thank you for your interest in contributing to MovieMate! This document provides guidelines and instructions for contributing.

## Getting Started

1. Fork the repository
2. Clone your fork
3. Set up the development environment (see SETUP.md)
4. Create a feature branch

## Development Workflow

1. **Create a feature branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```

2. **Make your changes**
   - Follow the existing code style
   - Write clear, readable code
   - Add comments where necessary

3. **Test your changes**
   - Test backend: `python manage.py test`
   - Test frontend: Manual testing in browser
   - Ensure no console errors

4. **Commit your changes**
   ```bash
   git add .
   git commit -m "Add: Description of your feature"
   ```

5. **Push to your fork**
   ```bash
   git push origin feature/your-feature-name
   ```

6. **Create a Pull Request**
   - Provide a clear description
   - Reference any related issues

## Code Style

### Python (Backend)
- Follow PEP 8 guidelines
- Use meaningful variable names
- Add docstrings to functions and classes
- Keep functions focused and small

### JavaScript/React (Frontend)
- Use functional components with hooks
- Follow ESLint rules
- Use meaningful variable names
- Keep components small and focused
- Use prop-types where applicable

## Commit Message Guidelines

Use clear, descriptive commit messages:
- `Add: New feature description`
- `Fix: Bug description`
- `Update: Change description`
- `Remove: Feature removal description`
- `Docs: Documentation update`

## Pull Request Guidelines

1. **Clear Title**: Summarize the changes
2. **Description**: Explain what and why
3. **Screenshots**: Include if UI changes
4. **Testing**: Describe how you tested
5. **Related Issues**: Reference any related issues

## Feature Ideas

We welcome contributions for:
- New features
- Bug fixes
- Performance improvements
- Documentation improvements
- UI/UX enhancements
- Test coverage
